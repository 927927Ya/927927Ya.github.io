前置准备

硬件要求

  • 操作系统：Windows / macOS / Linux（本教程以 Windows 为例）
  • Docker Desktop 已安装并正常运行
  • 至少 4GB 可用内存
  • 20GB 可用磁盘空间

软件要求

软件 版本要求 用途
Docker Desktop 最新版 运行容器
Dify 1.13.3 工作流引擎（需已部署）
通义千问 API Key 有效 视觉识别

你需要先有的东西

  1. Dify 已通过 Docker 部署并能通过 http://localhost 访问
  2. 通义千问 API Key（在 https://dashscope.aliyuncs.com 申请）
  3. 一个测试视频（MP4 格式，30 秒以内为宜）

如果还没部署 Dify，先按官方文档部署：https://github.com/langgenius/dify

附录 3：常见问题排查

问题 1：HTTP 节点超时

现象：工作流运行报错 “Reached maximum retries”

原因：视频太长或分镜太多，处理时间超过了 HTTP 节点的超时设置

解决：

  1. 编辑 HTTP 请求节点
  2. 在”超时设置”中增加 最大读取超时 到 600 秒以上
  3. 或者上传更短的视频（30 秒以内）

问题 2：容器启动报 SyntaxError

现象：docker logs 显示 SyntaxError: invalid syntax

原因：server.py 中的字符串包含中文引号（" "）或特殊字符，被 Python 解释为字符串结束

解决：

  • 确保所有引号使用 ASCII 引号（" 和 '）
  • 不要在字符串中使用反斜杠加汉字（如 \万岁），这会被解释为转义

问题 3：ffprobe 返回 404 / 文件找不到

现象：日志显示 download status=404

原因：Dify 生成的文件签名 URL 在容器内部网络无法验证

解决：确保 docker-compose 中已挂载共享存储：

问题 4：图片显示 [Image blocked]

现象：Dify 输出中图片位置显示 “[Image blocked]”

原因：Dify 的 Markdown 渲染器阻止了 data: URI 的图片

解决：不要用 base64 嵌入，改用 nginx 代理方式：

  • 确认 nginx 模板已添加 /shots/ 转发
  • 重启 nginx：docker compose restart nginx
  • 确认图片 URL 以 /shots/ 开头

问题 5：LLM 429 限流

现象：describe_shot 返回 “API错误:429”

原因：通义千问 API 调用过于频繁

解决：

  • 已内置 0.5 秒延迟，可以增加到 1-2 秒
  • 或者降低分镜数量（调高 SCENE_THRESHOLD）

问题 6：沙箱报错 “operation not permitted”

现象：工作流执行时报操作不允许

原因：Dify 沙箱的 seccomp 规则限制了系统调用

附录 4：参数调优指南

场景检测灵敏度

值 分镜数（30秒视频） 适用场景
0.4 约 8-12 个 访谈、教学、慢节奏
0.25 约 15-25 个 普通视频、Vlog
0.1 30+ 个 快节奏混剪、预告片

关键帧质量

  • KEYFRAME_QUALITY：值越小质量越高，但文件也越大。10 是平衡点
  • KEYFRAME_WIDTH：宽度越大细节越多，但 base64 体积也越大。512px 已足够 AI 识别
  • 图片质量会影响 LLM 的识别准确率，但太大会拖慢传输速度

API 调用参数

  • temperature：0.1-0.5 适合结构化输出，0.8+ 适合创意写作
  • max_tokens：1024 对于短描述足够了，不需要更大
