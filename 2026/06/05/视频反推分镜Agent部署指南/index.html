上传视频 → 自动拆分为多个分镜 → 生成 AI 描述 → 写入飞书多维表格（含关键帧截图）

1. 项目概述

整体架构

为什么不在 Dify 里直接处理？

限制 说明
Dify 沙箱禁止 subprocess 无法运行 FFmpeg
沙箱无法写临时文件 tempfile 权限问题
变量有 200KB 限制 base64 图片会被截断

解决方案： 把 FFmpeg + LLM 调用 + 飞书写入全部封装进独立 Docker 容器，Dify 只做一次 HTTP 调用。

飞书开放平台 企业自建应用

8. 常见问题排错

8.1 HTTP 节点超时

  • 原因： 视频太长或分镜数太多，处理时间超过默认超时
  • 解决： 将 HTTP 节点的超时设为 600s+

8.2 LLM 返回 429 限流

  • 原因： 连续请求过于频繁
  • 解决： 代码中已内置 0.5s 延迟，仍出现可增加到 1s

8.3 图片无法上传到飞书

  • 原因 1： 应用缺少 drive:file 权限
  • 原因 2： 关键帧截图列类型不是”附件”
  • 原因 3： 版本未发布导致权限不生效

排查： 检查 ffmpeg-server 日志 docker logs docker-ffmpeg-server-1 --tail 30

8.4 “params error” 错误

  • 原因： Drive 上传接口 multipart 参数名不对
  • 解决： 参数名用 size 不要用 file_size，手动拼接 multipart body

8.5 “FieldNameNotFound” 错误

  • 原因： 飞书表格列名不匹配
  • 解决： 确保 8 个列名完全一致：分镜编号、时间范围、画面主体、动作、环境、运镜、AI描述、关键帧截图

8.6 Dify 变量截断

  • 原因： Dify 变量有 200KB 上限
  • 解决： 大尺寸数据（base64 图片）在 ffmpeg-server 内部处理，不经 Dify 变量传递
