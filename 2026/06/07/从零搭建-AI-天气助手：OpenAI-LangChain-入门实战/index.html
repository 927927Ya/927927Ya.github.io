前言

本文会带你从零搭建一个 AI 天气助手：

  • 用自然语言问天气，AI 自动调用天气 API
  • 支持多轮对话，记得上下文
  • 有 Web 聊天界面，流式输出打字机效果
  • 从原生 OpenAI SDK 逐步过渡到 LangChain

全程使用 阿里云百炼 的兼容接口（国内可直连，无需翻墙）。所有代码都在一个项目里，每步可独立运行。

七、踩坑合集

做这个项目过程中遇到的所有坑，一次列全：

# 问题 原因 解决
1 请求一直超时 config.baseUrl 传给了参数 baseURL（大小写不匹配），请求打到 api.openai.com 记住 SDK 参数是 baseURL 大写
2 流式没有 usage DashScope 兼容 API 的限制 非流式才有统计
3 模型回复 null 模型调工具时 content 为空 判断 message.tool_calls 分支
4 多轮不记忆 每次都新建数组 数组放函数外部，.push() 追加
5 天气 API 卡死 fetch 没设超时 AbortSignal.timeout(5000)
6 思考过程刷屏 reasoning_content 被当普通 token 输出 只显示”思考中…”
7 模板语法错误 用了 new ChatPromptTemplate({...}) 用 fromMessages([...])
8 模板变量不生效 用了 {{role}} 双花括号 单花括号 {role}
9 前端打字机漏字 SSE 消息被 HTTP chunk 截断，最后一行不完整却被清空 用 lines.pop() 保留未完整接收的消息
10 多个工具结果互相覆盖 每个 tool_call 单独发请求，后面的看不到前面的结果 先收集所有 tool 结果，再一次性发给模型

总结

两条路线对比

  • OpenAI SDK：贴近底层，灵活，适合简单场景
  • LangChain：模板、链式、记忆、流水线等抽象，适合复杂流程

完整调用链路

这篇文章的代码都是以 .mjs 文件形式提供的，可以直接 node xxx.mjs 运行。建议跟着顺序从 01-basic.mjs 到 04-langchain.mjs 逐步实验，每一步都是下一节的基础。祝编码愉快！
